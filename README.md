# Прогнозирование цен на автомобили (в рамках дз 1 по ML HSE)

## Ссылка на приложение: https://mlhsehw1autocostpredictionservice-ez4kcmltrogk4w4dz7uuyh.streamlit.app/

## Структура проекта
- `app.py` - Streamlit приложение
- `best_ridge_model.pkl` - обученная модель Ridge
- `preprocessing.pkl` - пайплайн предобработки данных  
- `notebook.ipynb` - полный анализ и эксперименты
- `README.md` - документация
- `data_test.csv` - данные для тестирования работы
- `requirements.txt` - спиок библиотек для работы проекта

## Что было сделано

### EDA и предобработка
Анализ данных на пропуски и дубликаты. Преобразование строковых признаков в числовые. Разделение сложных признаков на составляющие.

### Генерация признаков
Создание новых признаков на основе имеющихся. Добавление категориальных признаков через One-Hot Encoding. Стандартизация числовых данных.

### Обучение моделей
Обучение линейных моделей с регуляризацией. Подбор гиперпараметров. Создание кастомных бизнес-метрик.

## Результаты

### Метрики
Лучшая модель показала высокие значения R² на тестовых данных.

### Важные признаки
Наиболее значимыми оказались технические характеристики автомобиля и категориальные признаки. Мощность двигателя, год выпуска и страна производства - ключевые факторы цены.

## Наибольший прирост качества

Добавление категориальных признаков дало максимальное улучшение качества. Ridge регуляризация показала лучшие результаты по сравнению с другими методами.

## Что не удалось

Не все форматы записи удалось корректно обработать из-за разнообразия исходных данных.
Не реализованы все возможные преобразования признаков из-за ограничений по времени.
Упрощенный подход к обработке экстремальных значений.

## Оценка сервиса

### Удобство
Интерфейс с 4 вкладками: анализ, загрузка CSV, ручной ввод, модель. Все просто и интуитивно понятно. Красиво реализовал ручной ввод параметров. 

### Визуализации
Отобразил основные информативные графики: графики зависимостей, матрица корреляций, важность признаков. Можно сразу смтореть предсказания и загружаемые данные.

### Проблемы
- Нет некоторых обработок ошибок при вводе данных
- Специфичный формат ввода torque
- Теряется скорость работы при построении большого количества графиков, из-за чего от части пришлось отказаться.

### Планируемые улучшения
Больше интерактивных графиков, полная обработка возможных ошибок при вводе значений пользователем.
